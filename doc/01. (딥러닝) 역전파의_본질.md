### 1단계: 무작위 텐서 배열의 탄생 (Initialization)

> _"처음엔 기준 없이, 아무 값이나 랜덤으로"_

딥러닝 모델의 뇌(가중치)는 처음에 합이 1이어야 한다는 제약 없이, 자유분방한 랜덤 숫자들의 텐서(Tensor)로 메모리에 적재됩니다.

Python

```
# 제한 없는 날것의 가중치 텐서
weights = torch.tensor([0.12, -0.45, 0.89, ...], requires_grad=True)
```

### 2단계: 예측과 포장 (Forward & Softmax)

> _"합이 1이 아니지만, 결국 1인 확률 텐서로 변환된다"_

입력 데이터가 이 무작위 가중치들과 곱해져 거친 원본 점수(Logit)를 만들고, 이를 **Softmax**에 통과시키면 전체 분모가 출렁이며 **합이 1인 확률 텐서**로 예쁘게 포장됩니다.

Python

```
# 1. 내부 연산을 거친 원본 점수 (합이 1 아님)
logits = torch.tensor([2.5, 0.1, 5.2]) 

# 2. Softmax를 거친 최종 확률 예측값 (합이 1)
probs = torch.tensor([0.06, 0.01, 0.93]) 
```

### 3단계: 정답 텐서와의 만남과 기울기 추출 (Loss & Gradient) ★핵심★

> _"target은 인덱스! 이 인덱스를 기준으로 $p - y$ 뺄셈이 일어나 기울기 텐서가 만들어진다"_

시스템은 `target` 텐서를 통해 "이번 정답은 0번 인덱스야"라는 지시를 받습니다. 이때 내부적으로 오답 클래스의 정답 확률($y$)은 **0**, 정답 클래스의 확률($y$)은 **1**로 취급하여 **기울기($p - y$) 텐서**를 순식간에 계산해 냅니다.

Python

```
target = torch.tensor([0]) # 정답은 0번 인덱스

# 시스템 내부의 마법의 뺄셈 (Gradient = p - y)
# probs: [0.06, 0.01, 0.93]
# y(정답): [1,    0,    0   ] (0번 인덱스만 1)

# 기울기 텐서 결과:
gradients = torch.tensor([-0.94,  0.01,  0.93])
```

- **정답 (0번):** $0.06 - 1 = \mathbf{-0.94}$ (강한 음수 $\rightarrow$ 배후 가중치들에게 **"점수 팍팍 밀어 올려(뻥튀기)!"** 명령)
    
- **오답 (1번):** $0.01 - 0 = \mathbf{0.01}$ (약한 양수 $\rightarrow$ **"점수 조금 깎아!"**)
    
- **오답 (2번):** $0.93 - 0 = \mathbf{0.93}$ (강한 양수 $\rightarrow$ 분모를 불린 주범이므로 **"점수 크게 깎아내려!"**)
    

### 4단계: 랜덤 값들 사이의 무한 재조정 (Readjustment)

> _"랜덤한 값들 내에서 기울기 압력을 받으며 서로 밸런스를 맞춘다"_

옵티마이저(Optimizer)는 저 기울기 텐서(`[-0.94, 0.01, 0.93]`)를 가이드 삼아 처음의 그 무작위 가중치 텐서들을 수정합니다.

오답을 만든 가중치들은 서로 깎이고, 정답을 맞힌 가중치들은 뻥튀기됩니다. 이 과정을 수만 번 반복하면, 처음에 제멋대로 놀던 랜덤 값들이 **확률 100% (기울기 0)를 향해 스스로를 끝없이 재조정(Readjust)하며 가장 안정적인 상태로 굳어집니다.**

### 5단계: 실전 투입 (Inference & Argmax)

> _"최종적으로 재조정된 텐서 중 가장 높은 값(인덱스)을 반환"_

학습이 모두 끝나고 나면, AI는 더 이상 정답지(`target`)를 보지 않고 뺄셈도 하지 않습니다.

완벽하게 밸런스가 맞춰진 가중치들을 통과해 나온 최종 점수 텐서에 `argmax`를 때려서 1등을 한 인덱스를 반환합니다.

Python

```
# 실전 예측
final_logits = model(new_data)
# 가장 점수가 높은 인덱스 추출
prediction = torch.argmax(final_logits) 
```

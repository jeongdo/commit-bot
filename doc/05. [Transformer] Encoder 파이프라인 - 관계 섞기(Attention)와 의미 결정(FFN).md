# [Transformer] Multi-Head Attention : 8차원을 쪼개는 다각도의 시선

> ChatGPT와의 대화 내용 기반 정리 (통합본)

---

## 1. Q, K, V 기본 구조

입력 토큰마다 Q, K, V 세 가지 벡터를 **동시에** 생성한다.

예시 입력: `버그 수정 완료`

| 토큰 | Q | K | V |
|------|---|---|---|
| 버그 | Q버그 | K버그 | V버그 |
| 수정 | Q수정 | K수정 | V수정 |
| 완료 | Q완료 | K완료 | V완료 |

### 중요한 착각 정리

```
❌ "Q 만든 다음 K 만들고 V 만든다"  → 순차 단계가 아님

✅ 실제: X → (Wq, Wk, Wv) 동시에 통과 → Q, K, V 생성
```

**역할 구분**

| 역할 | 의미 |
|------|------|
| Q | 비교 기준 ("질문") |
| K | 비교 대상 |
| V | 실제 가져올 값 (원본 정보) |

---

## 2. Self-Attention 연산 순서

```
1. X (embedding + PE)
2. Q/K/V 생성 (동시에)
3. Q·K 내적 → Attention Score
4. Softmax → 가중치 비율
5. V 가중합 → 새 벡터 생성
```

### 2-1. N×N 관계 매트릭스

각 토큰의 Q는 **문장 전체의 K** (자신 포함)와 모두 비교한다.

```
문장 길이 3 기준 → 3×3 매트릭스

         K버그  K수정  K완료
Q버그      ○      ○      ○
Q수정      ○      ○      ○
Q완료      ○      ○      ○
```

→ 자기 자신도 포함해서 비교하기 때문에 이름이 **Self** Attention

### 2-2. 버그 토큰 기준 예시

**Step 1. Attention Score (내적)**
```
Q버그 · K버그 = 2.1
Q버그 · K수정 = 1.4
Q버그 · K완료 = 0.9
```

**Step 2. Softmax (비율 변환)**
```
[2.1, 1.4, 0.9]  →  [0.5, 0.3, 0.2]
```

**Step 3. V 가중합**
```
0.5 × V버그
+ 0.3 × V수정
+ 0.2 × V완료
= 새로운 버그 벡터
```

> ✅ **업데이트 대상은 Q도 V도 아니라, 해당 위치의 출력 벡터**
> ✅ **V는 계산 결과가 아니라 원본 정보**

## 3. Wq, Wk 학습 원리

- 초기에는 **랜덤 가중치**로 시작
- 수만 건의 학습 데이터를 반복하면서 Wq, Wk가 조정됨

```
버그 수정 완료  →  fix bug completed
버그 수정       →  fix bug
```

→ 학습 결과: `Q버그 · K수정` 점수가 자연스럽게 높아짐  
→ "관계를 알고 있어서" 높은 게 아니라, **학습으로 Wq/Wk가 변한 것**

## 4. Attention만 있을 때의 한계

현재 구조 (FFN 없음):

```
Embedding → PE → Self-Attention → Linear
```

**결과:**
```
버그 = (버그 + 수정 정보가 섞인 벡터)  ← "혼합 벡터"
수정 = (수정 + 버그 정보가 섞인 벡터)  ← "혼합 벡터"
```

→ 역할 분리(문제/해결)가 아직 생기지 않은 상태

---

## 5. FFN의 역할

| 구조 | 결과 |
|------|------|
| Attention만 | 서로 섞인 벡터 ("혼합") |
| Attention + FFN | 의미가 정리된 벡터 ("결정") |

```
❌ attention = 의미 완성    → 아님
❌ FFN = 단순 보조          → 아님

✅ attention = 관계 섞기
✅ FFN       = 의미 정제 / 번역 결과 확정
```

### 커밋 번역에서 FFN이 구분하는 것

```
bug fix    vs  bug patch
refactor   vs  restructure
code update vs  code cleanup
```

→ **"문맥을 하나의 결정으로 압축"**

## 6. 완성된 Encoder 구조

```
Embedding
    ↓
Self-Attention  ← 관계 섞기
    ↓
FFN             ← 의미 결정
    ↓
Output
```
